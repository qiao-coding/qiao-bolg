# Markdown 结构目录

## 1. 基础语法

### 1.1 标题
```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### 1.2 段落和换行
```markdown
这是第一段。

这是第二段，段落之间需要空行分隔。

如果需要换行但不换段落，
可以在行尾添加两个空格  
然后继续写
```

### 1.3 强调
```markdown
*斜体* 或 _斜体_
**粗体** 或 __粗体__
***粗斜体*** 或 ___粗斜体___
~~删除线~~
```

### 1.4 列表

#### 1.4.1 无序列表
```markdown
- 项目一
- 项目二
  - 子项目
  - 子项目
- 项目三
```

#### 1.4.2 有序列表
```markdown
1. 第一项
2. 第二项
   1. 子项
   2. 子项
3. 第三项
```

#### 1.4.3 任务列表
```markdown
- [x] 已完成的任务
- [ ] 未完成的任务
- [x] 另一个已完成的任务
```

## 2. 高级语法

### 2.1 链接
```markdown
[链接文字](https://example.com)
[链接文字](https://example.com "可选标题")
<https://example.com> (自动链接)
[引用链接][id]
[id]: https://example.com "链接标题"
```

### 2.2 图片
```markdown
![替代文字](图片路径 "可选标题")
![替代文字](图片路径)
![替代文字][图片引用]
[图片引用]: 图片路径 "标题"
```

### 2.3 代码
```markdown
行内代码: `code`
代码块:
```
这是一个代码块
```

指定语言的代码块:
```javascript
console.log('Hello World');
```
```

### 2.4 引用
```markdown
> 这是一个引用
> 可以跨多行
>> 还可以嵌套引用

普通文本

> 引用中可以包含其他 Markdown 语法
> 
> 1. 列表
> 2. `代码`
> 3. [链接](https://example.com)
```

### 2.5 分割线
```markdown
---
***
___
```

## 4. 扩展语法 (GFM - GitHub Flavored Markdown)

### 4.1 表情符号
```markdown
:smile: :thumbsup: :rocket: :bug: :warning:
```

### 4.2 脚注
```markdown
这是一个脚注示例[^1]。

[^1]: 这是脚注的内容
```

### 4.3 定义列表
```markdown
术语 1
: 定义 1

术语 2
: 定义 2
  带有更多细节的定义
```

### 4.4 任务列表 (在 GitHub 中)
```markdown
- [x] 任务一
- [ ] 任务二
- [x] 任务三
```

## 5. HTML 元素

### 5.1 内联 HTML
```markdown
这是 <em>强调</em> 的文本，也可以使用 <strong>strong</strong>。
```

### 5.2 块级 HTML
```markdown
<div class="note">
  <p>这是嵌入的 HTML 块</p>
</div>
```

## 10. 最佳实践

### 10.1 文件结构
```
文章标题.md
├── 标题层级 (h1-h6)
├── 内容段落
├── 列表和表格
├── 代码块
├── 引用
└── 链接和图片
```

### 10.2 写作建议
- 使用一致的标题层级
- 保持段落简洁
- 合理使用列表组织内容
- 为图片添加有意义的替代文字
- 使用有意义的链接文字

### 10.3 可读性建议
- 在行尾添加两个空格来强制换行
- 使用空行分隔段落和块级元素
- 保持代码块的缩进一致
- 为复杂表格添加表头
